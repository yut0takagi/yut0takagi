# はじめに

macOS 標準の `/usr/bin/ruby` ではなく、rbenv でインストールした Ruby を使いたいのに、

```console
rbenv global 3.2.8
rbenv rehash
ruby -v
# => ruby 2.6.10p210 (…)
```

のように **バージョンが変わらない**──そんな経験はありませんか？
さらに fish シェルで誤って zsh 用設定を `source ~/.zshrc` した結果、

```console
~/.zshrc (line 12): Unexpected ')' found, expecting '}'
```

というエラーに遭遇してハマったので、**原因と解決策**をまとめました。

---

## 環境

| 項目       | 内容                              |
| -------- | ------------------------------- |
| macOS    | Sonoma 14.x (Apple Silicon)     |
| シェル      | fish 3.9.x                      |
| Ruby     | 3.2.8 (rbenv) / 2.6.10 (System) |
| rbenv    | 1.3.3                           |
| Homebrew | /opt/homebrew                   |

---

## 症状

1. `rbenv global` でバージョンを切り替えても **`ruby -v` が 2.6.10** のまま。
2. `which ruby` が `/usr/bin/ruby` を指す。
3. zsh 用ファイルを `source` して fish が **「Unexpected ')'」** と怒る。

---

## 原因

* **fish で zsh 用設定を読み込んだ** → 文法が違うためシンタックスエラー。
* `set -x PATH …` で **PATH を決め打ち** → `~/.rbenv/shims` が消滅。
* `brew shellenv` を PATH 設定の後に呼び、**PATH が再構築** → rbenv 無効化。

---

## 解決策

### 1 fish 用 `config.fish` を正しく書く

`~/.config/fish/config.fish` を以下に統一します。

```fish
# -------------------------------
# Homebrew (最初に呼び出す)
# -------------------------------
eval (/opt/homebrew/bin/brew shellenv)

# -------------------------------
# rbenv
# -------------------------------
set -gx RBENV_ROOT $HOME/.rbenv
set -gx PATH $RBENV_ROOT/bin $PATH
status --is-interactive; and rbenv init - fish | source

# -------------------------------
# 追加ツール
# -------------------------------
fish_add_path $HOME/.local/bin
fish_add_path $HOME/.lmstudio/bin
fish_add_path $HOME/.codeium/windsurf/bin
```

> **ポイント**
>
> 1. **Homebrew を最初**に呼ぶ。`brew shellenv` は PATH を作り直すため。
> 2. 直後に **rbenv** を初期化し、`~/.rbenv/shims` を PATH 先頭へ。
> 3. 追加ツールは `fish_add_path` で **末尾に足す** と安全。

### 2 設定を反映

```console
source ~/.config/fish/config.fish  # またはターミナル再起動
which ruby                         # => ~/.rbenv/shims/ruby
ruby -v                            # => ruby 3.2.8p...
```

### 3 GUI アプリにも適用（任意）

```console
sudo launchctl config user path \
"$HOME/.rbenv/shims:$HOME/.rbenv/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin"
# ⇒ ログアウト / ログイン
```

### 4 チェックリスト（まだ直らない場合）

| チェック項目           | コマンド                                               |
| ---------------- | -------------------------------------------------- |
| rbenv が複数ないか     | `which -a rbenv`                                   |
| PATH を再上書きしていないか | `grep -n "set -x PATH" ~/.config/fish/config.fish` |
| コマンドキャッシュ        | `hash -r`                                          |
| Ruby が壊れていないか    | `rbenv uninstall 3.2.8 && rbenv install 3.2.8`     |

---

## まとめ

* **fish なら `config.fish` 一箇所に集約**し、PATH の順序を死守する。
* `brew shellenv` → `rbenv init - fish` → その他 PATH 追加の順が鉄則。
* GUI でも統一したい場合は `launchctl config user path` を設定。

同じように “`ruby -v` が変わらない” で悩む方の助けになれば幸いです。
