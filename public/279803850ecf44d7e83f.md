## 🎯 対象読者

* Reactで作ったポートフォリオなどを無料で公開したい方
* VercelやNetlify以外の選択肢としてRenderを使ってみたい方

## 🛠️ 前提条件

* GitHubにReactプロジェクトをPush済み
* プロジェクト構造は以下のようになっている：

```
Portfolio/
├── cliant/                ← Reactアプリのルート
│   ├── public/
│   ├── src/
│   ├── package.json
│   └── ...
└── README.md
```

※ `cliant` は `client` のタイプミスと思われるが、そのまま進行可。

---

## 🪜 手順

### ① RenderでStatic Siteを作成

1. [https://dashboard.render.com/](https://dashboard.render.com/) にアクセスしログイン
2. 「New」 → 「Static Site」を選択
3. GitHubリポジトリにRenderを接続し、対象のリポジトリ（例：Portfolio）を選択
4. 各種設定を入力：

| 設定項目                  | 値                              |
| --------------------- | ------------------------------ |
| **Name**              | 任意（例：Portfolio）                |
| **Root Directory**    | `cliant`                       |
| **Build Command**     | `npm install && npm run build` |
| **Publish Directory** | `build`                        |

5. 「Create Static Site」をクリック

---

### ② デプロイの確認

* ビルドが成功すれば `your-project.onrender.com` のURLでアクセス可能になります
* Pushするたびに自動デプロイされます

---

## 🧼 よくあるエラーと対処法

### ❌ ENOENT: package.json not found

* 原因：ルートディレクトリが `cliant` でない
* 対策：**Root Directory に正しく `cliant` を指定すること！**

### ❌ Start Command のエラー

* 原因：**Static Siteでは不要なStart Commandが設定されている**
* 対策：Web Serviceではなく**Static Site**として作成しなおす

---

## 🔒 注意点

* Reactはビルド後はただのHTML/CSS/JSなので、**Static SiteでOK**
* 認証やサーバーサイド処理がある場合は、Web Service や Flask/Node.js の別途構成が必要

---

## ✅ まとめ

| 項目                | 内容                             |
| ----------------- | ------------------------------ |
| Render種別          | Static Site                    |
| Build Command     | `npm install && npm run build` |
| Publish Directory | `build`（`cliant/build` 内部）     |
| 自動デプロイ            | GitHubのPushで反映                 |

## ✨ おすすめ

* Tailwind CSSやVanta.jsと組み合わせて印象的なポートフォリオに！
* `render.yaml` でインフラをIaC管理するのも便利（応用編）
