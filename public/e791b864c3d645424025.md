以前から、何度も発生していたエラーについての改善です。

## 発生する現象

Create React App 環境で Tailwind CSS を設定しようとすると、以下のようなエラーが発生することがある。

```bash
npx tailwindcss init -p
sh: tailwindcss: command not found
```

また、`npm start`時に以下の様なエラー:

```bash
Error: It looks like you're trying to use `tailwindcss` directly as a PostCSS plugin. The PostCSS plugin has moved to a separate package...
```

## 原因

Tailwind CSS v4.x 以降で PostCSS プラグインが分離され、Create React App (CRA) の PostCSS v8.x 構成との互換性に問題が発生。

## 解決第1手: Tailwind CSS v3.x にダウングレード

### ステップ 1. 旧バージョンの削除

```bash
npm uninstall tailwindcss postcss autoprefixer @tailwindcss/postcss
```

### ステップ 2. 安定版の再インストール

```bash
npm install -D tailwindcss@3.4.3 postcss@8.4.31 autoprefixer@10.4.14
```

### ステップ 3. postcss.config.js

```js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

### ステップ 4. tailwind.config.js

```js
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

### ステップ 5. src/index.css

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### ステップ 6. npm start

```bash
npm start
```

## ポイント

* Tailwind v4以降は Vite / Next.js での利用を前提
* Create React App では v3.x を利用するのが安定

## おまけ: Vite + React にするとめちゃくちゃ楽

```bash
npm create vite@latest my-portfolio -- --template react
cd my-portfolio
npm install
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```
