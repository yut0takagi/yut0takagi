## TL;DR

* **fish を使っていると `export` が効かない** → `set -Ux EDITOR "<editor> --wait"`
* VS Code / Cursor など“フォークして戻る”エディタは **`--wait`** が必須
* zsh/bash なら `export EDITOR="<editor> --wait"`

---

## 開発環境

| ソフト    | バージョン                |
| ------ | -------------------- |
| macOS  | 15.5 (Apple Silicon) |
| Ruby   | 3.3.0                |
| Rails  | 7.1.3                |
| fish   | 3.7.1                |
| Cursor | 0.30                 |

> **注:** どのエディタでも手順は同じ。VS Code や Vim ならコマンドだけ変えてください。

---

## エラー再現

```bash
$ rails credentials:edit
No $EDITOR to open file in. Assign one like this:

EDITOR="mate --wait" bin/rails credentials:edit
```

* **`config/credentials.yml.enc` を削除したあと**に出やすい。
* Rails は `ENV["EDITOR"]` または `ENV["VISUAL"]` を読んでエディタを決定。
* fish で `export` は使えない → Rails から見ると **未設定** になる。

---

## 原因まとめ

1. fish の文法が bash/zsh と違う（`export VAR=value` が存在しない）。
2. GUI エディタはプロセスを fork してすぐ戻るため、`--wait` がないと Rails が「編集終わり」と誤判定。

---

## 解決策（fish）

### ① 1 回だけ試す

```bash
set -gx EDITOR "cursor --wait"   # 現在のシェルだけ
set -gx VISUAL $EDITOR
rails credentials:edit            # 動くか確認
```

### ② 永続化する（推奨: ユニバーサル変数）

```bash
# “ユーザ全体”で永続的に保持
set -Ux EDITOR "cursor --wait"
set -Ux VISUAL $EDITOR
```

* `-U` は **ユニバーサル変数**: 次回以降のセッションでも自動で有効。
* 設定後すぐ有効になるので `source` や再起動は不要。

### ③ `config.fish` に書く場合（明示的に残したいとき）

```fish
# ~/.config/fish/config.fish
set -gx EDITOR "cursor --wait"
set -gx VISUAL $EDITOR
```

保存後:

```bash
source ~/.config/fish/config.fish
```

---

## 解決策（zsh/bash）

```bash
# ~/.zshrc など
export EDITOR="cursor --wait"
export VISUAL="$EDITOR"
source ~/.zshrc        # 反映してから
rails credentials:edit
```

> *fish 上で `source ~/.zshrc` を実行すると文法エラーになるので注意！*

## よくあるハマりポイント

| 症状                          | 対処                                                                               |
| --------------------------- | -------------------------------------------------------------------------------- |
| `cursor: command not found` | Cursor CLI をインストール（コマンドパレット → *Shell Command: Install 'cursor' command in PATH*） |
| 何も書かずに閉じたら空のままコミット          | `git checkout -- config/credentials.yml.enc` でロールバックし再編集                         |
| `master.key` 紛失             | 本番で復号できない。`RAILS_MASTER_KEY` を再生成 + デプロイ環境に設定                                    |

---

## まとめ

* **fish では `set -Ux`、zsh/bash では `export`** で `$EDITOR` / `$VISUAL` を定義。
* GUI エディタは `--wait` オプションが必須。
* 誤って `credentials.yml.enc` を削除した場合は、再生成後に各種シークレットを入れ直す。

これで *“No \$EDITOR to open file in”* 問題は解決です！ 💡
